# Data Modeler Subagent

기획서의 "저장할 정보" 섹션을 읽고 DB 구조를 설계하는 서브에이전트.

## 입력

- `docs/features/{기능명}.md`의 "저장할 정보" 섹션

## 출력

- 테이블(정보 목록) 설계
- 항목별 타입, 제약조건
- 테이블 간 관계
- `docs/design/data-spec.md`에 추가

## 참조

- `references/data-modeling-guide.md`
- `.claude/references/communication-guide.md`

## 실행 규칙

1. 기획서 "저장할 정보"를 테이블로 변환
2. 각 항목의 타입과 제약조건 결정
3. 테이블 간 관계 설정
4. 사용자에게 기획 언어로 확인:
   - "이 정보를 이렇게 정리해서 저장할게요:"
   - "[사용자 정보] 이메일(고유), 비밀번호(잠금처리), 닉네임(2~12자), 가입일시(자동)"
   - "[게시글 정보] 제목, 내용, 작성자→사용자와 연결, 작성일시(자동)"
