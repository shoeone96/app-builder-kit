# 기술 ↔ 기획 용어 매핑

## 코드 패턴 → 기획 언어

| 코드 | 기획 언어 |
|------|----------|
| `if (condition)` | "만약 ~이면" |
| `throw new Error` | "~라고 알려줘요" / "문제가 생겨요" |
| `try { } catch { }` | "이걸 시도하고, 문제가 생기면 이렇게 해요" |
| `await fetch(url)` | "서버에 ~를 요청해요" |
| `return data` | "~를 돌려줘요" |
| `for / map` | "하나씩 반복해서" |
| `filter` | "조건에 맞는 것만 골라서" |
| `sort` | "순서대로 정리해서" |
| `.findUnique()` | "~를 찾아요" |
| `.findMany()` | "목록을 가져와요" |
| `.create()` | "새로 만들어요" |
| `.update()` | "수정해요" |
| `.delete()` | "삭제해요" |
| `@unique` | "고유 (중복 불가)" |
| `@default(now())` | "자동으로 기록" |
| `hashPassword()` | "비밀번호를 잠금 처리해요" |
| `comparePassword()` | "비밀번호가 맞는지 확인해요" |

## 파일 역할 → 기획 언어

| 파일 패턴 | 기획 언어 |
|----------|----------|
| controller / route | "요청을 받는 곳" |
| service | "실제 처리하는 곳" |
| model / entity | "정보 구조 정의" |
| middleware | "중간에 확인하는 곳 (로그인 여부 등)" |
| schema.prisma | "저장할 정보 구조" |
| page.tsx | "화면" |
| component | "화면 조각" |
| hook | "재사용 기능" |
| utils / lib | "도구 모음" |

## 토스 OSS → 기획 언어

| 코드 | 기획 언어 |
|------|----------|
| `overlay.open()` | "팝업/안내창을 열어요" |
| `overlay.close()` | "팝업을 닫아요" |

| `useFunnel()` | "단계별로 하나씩 입력받아요" |
| `debounce()` | "연속 입력을 기다렸다가 마지막만 처리해요" |
| `groupBy()` | "같은 종류끼리 묶어요" |
| `sortBy()` | "순서대로 정렬해요" |

## 번역 패턴

### 동작 흐름
```
코드: fetchUser → validateInput → saveToDb → returnResponse
번역: "사용자 정보를 가져와서 → 입력값을 확인하고 → 저장소에 저장한 뒤 → 결과를 돌려줘요"
```

### 조건 분기
```
코드: if (exists) throw ConflictError; else create()
번역: "이미 있으면 '이미 존재해요'라고 알려주고, 없으면 새로 만들어요"
```

### 반복
```
코드: users.map(u => u.nickname)
번역: "사용자 목록에서 닉네임만 쏙 뽑아와요"
```
