# /설명해 — 기술 개념 쉽게 설명

모르는 것을 물어보면 쉽게 설명하는 스킬.

## 트리거

"이게 뭐야?", "무슨 뜻이야?", "왜 이래?", "설명해줘"

## 참조

- `.claude/subagents/logic-explainer/references/glossary.md`
- `.claude/references/communication-guide.md`

## 실행 규칙

1. 질문한 개념/현상 파악
2. glossary.md 참조해서 일상 비유로 설명
3. 비유 → 실제 의미 → 이 앱에서의 역할 순서

## 설명 패턴

### 개념 질문
```
사용자: "서버가 뭐야?"
응답: "식당으로 비유하면, 서버는 주방이에요.
      손님(앱)이 주문(요청)하면, 주방(서버)에서 요리(처리)해서 음식(결과)을 가져다줘요.
      이 앱에서는, 사용자가 로그인하면 서버가 비밀번호를 확인하고 결과를 알려줘요."
```

### 에러 질문
```
사용자: "이 에러가 뭐야?"
응답: "이건 '이미 사용 중인 이메일'이라는 뜻이에요.
      다른 사람이 이미 이 이메일로 가입해서, 같은 이메일로 또 가입할 수 없어요.
      다른 이메일을 쓰면 해결돼요."
```

### 파일 질문
```
사용자: "이 파일이 뭐야?"
응답: "이 파일은 로그인/회원가입 요청을 받는 곳이에요.
      사용자가 로그인 버튼을 누르면, 이 파일이 가장 먼저 그 요청을 받아요."
```

### "왜" 질문
```
사용자: "왜 비밀번호를 그냥 저장하면 안 돼?"
응답: "집 열쇠를 생각해보세요.
      열쇠를 그대로 현관 앞에 두면 누구나 들어올 수 있잖아요.
      그래서 비밀번호도 잠금 처리(암호화)해서 저장해요.
      누군가 저장소를 들여다봐도 원래 비밀번호를 알 수 없게요."
```
