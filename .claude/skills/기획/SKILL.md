# /기획 — 기능 기획

앱에 넣을 기능을 하나씩 기획하는 핵심 스킬.

## 트리거

"기능 추가", "이런 거 만들고 싶어", "기획", "새 기능"

## 참조

- `references/question-flow.md`
- `.claude/references/communication-guide.md`

## 실행 흐름

### Step 1: 기능 파악

"어떤 기능이에요? 한 줄로 설명해주세요."
- 예: "회원가입", "게시글 작성", "채팅"

### Step 2: 사용 상황

"이 기능을 쓰는 상황을 설명해주세요."
- 예: "처음 앱을 열었을 때 가입 버튼을 누르면 시작"
- 모르겠다면: "언제 이 기능을 쓰게 될까요? 처음 들어왔을 때? 특정 버튼을 눌렀을 때?"

### Step 3: 데이터 질문 3단계

`question-flow.md` 참조하여 순서대로:

**a. 표시 데이터**: "이 기능에서 사용자한테 뭘 보여줘요?"
- 목록? 상세 정보? 입력 폼?

**b. 입력 데이터**: "사용자가 뭘 입력하거나 선택해요?"
- 텍스트? 숫자? 선택? 이미지?
- 각 항목: 필수/선택, 길이 제한, 조건

**c. 저장 여부**: "입력한 건 나중에 또 쓰이나요?"
- 예 → 저장할 정보로 기록
- 아니오 → 일회성 처리

### Step 4: 예외 상황

"이 기능에서 뭔가 잘못될 수 있는 경우가 있나요?"
- 모르겠다면 일반적인 예외를 제안:
  - "중복 입력하면?"
  - "필수 항목을 안 채우면?"
  - "로그인 안 한 상태면?"

### Step 5: 기획서 정리 및 확인

기획서를 보여주고 "이렇게 정리했어요. 맞나요?"

### Step 6: 저장

`docs/features/{기능명}.md` 저장
`docs/progress.md` 업데이트: 📋 기획 예정 → 📋 기획 완료

## 기획서 출력 형식

```markdown
# 기능: {기능명}

## 한 줄 설명
{한 줄 설명}

## 사용 상황
{사용 상황}

## 보여줄 정보
- {항목 1}
- {항목 2}

## 입력받을 정보
- {항목} ({필수/선택}, {조건})

## 저장할 정보
- {항목} ({특이사항})

## 예외 상황
- {상황 1}: {대응}
- {상황 2}: {대응}
```
