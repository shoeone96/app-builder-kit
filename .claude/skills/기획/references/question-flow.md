# 기획 질문 흐름 가이드

## 질문 원칙

1. 한 번에 하나만 질문
2. "잘 모르겠어요" 응답 시 선택지 제안
3. 추상적 답변 → 구체화 질문 추가
4. 기술 용어 절대 사용 금지

## 질문 흐름도

```
기능 한 줄 설명
    ↓
사용 상황 (언제, 어떻게 시작?)
    ↓
보여줄 정보 (뭘 보여줘요?)
    ↓
입력받을 정보 (뭘 입력해요?)
    ├─ 각 항목: 필수? 길이? 조건?
    ↓
저장 여부 (나중에 또 써요?)
    ├─ 예 → 저장할 정보 정리
    └─ 아니오 → 넘어감
    ↓
예외 상황 (잘못될 수 있는 것?)
    ↓
정리 + 확인
```

## 구체화 질문 예시

### "보여줄 정보"가 모호할 때
- "목록으로 보여줘요, 아니면 상세 정보 하나를 보여줘요?"
- "다른 사람 것도 보여요, 본인 것만 보여요?"
- "사진이나 이미지도 같이 보여줘요?"

### "입력받을 정보"가 모호할 때
- "글자를 직접 쓰는 건가요, 선택하는 건가요?"
- "이 항목은 꼭 필요한 건가요, 없어도 되는 건가요?"
- "최대 몇 글자까지 쓸 수 있으면 좋을까요?"

### 예외 상황 제안 (일반적인 것들)
- 이미 있는 정보로 또 시도할 때 (중복)
- 필수 항목을 비워놓고 제출할 때
- 로그인 안 한 상태에서 접근할 때
- 권한 없는 사람이 접근할 때
- 서버에 문제가 있을 때

## 기능 유형별 추가 질문

### 회원가입 / 로그인
- "어떤 방법으로 가입할 수 있어요?" (이메일, 소셜 로그인)
- "비밀번호 규칙이 필요해요?" (길이, 특수문자)
- "이메일 인증이 필요해요?"

### 게시글 / 글 작성
- "글에 사진이나 파일을 첨부할 수 있어요?"
- "다른 사람이 이 글을 수정/삭제할 수 있어요?"
- "글 작성자 이름이 보여요?"

### 검색
- "어떤 정보로 검색해요?" (이름, 태그, 내용)
- "검색 결과를 어떤 순서로 보여줘요?" (최신순, 인기순)

### 알림
- "어떤 상황에서 알림이 와요?"
- "알림을 누르면 어디로 이동해요?"
