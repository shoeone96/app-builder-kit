# /바꿔 — 기존 기능 수정

이미 만들어진 기능을 수정하는 스킬.

## 트리거

"바꾸고 싶어", "수정", "이거 좀 다르게", "변경"

## 참조

- `.claude/references/communication-guide.md`
- subagent: logic-explainer

## 실행 흐름

### Step 1: 수정 대상 파악

"어떤 기능을 바꾸고 싶어요?"
- 구체적이면 바로 진행
- 모호하면 완성된 기능 목록 보여주기

### Step 2: 현재 동작 설명

logic-explainer subagent 호출:
- 해당 기능의 코드를 읽고 기획 언어로 번역
- "지금은 이렇게 돌아가고 있어요:"
  - "사용자가 A를 하면 → B가 되고 → C가 보여요"

### Step 3: 변경 요청 받기

"어떻게 바꾸고 싶어요?"

### Step 4: 영향 범위 설명

변경으로 인한 영향 분석:
- "이걸 바꾸면 이런 부분들이 같이 바뀌어요:"
  - "{파일/기능 1}: 이렇게 바뀜"
  - "{파일/기능 2}: 이렇게 바뀜"
- "이런 식으로 바꾸면 될 것 같은데, 괜찮을까요?"

### Step 5: 구현 (승인 후)

- 코드 수정
- 테스트 (가능한 경우)

### Step 6: 문서 업데이트

- `docs/features/{기능명}.md` 업데이트
- `docs/progress.md` 업데이트

### 완료

"수정 완료했어요! 바뀐 내용을 정리하면:"
- 변경 전 → 변경 후 비교
