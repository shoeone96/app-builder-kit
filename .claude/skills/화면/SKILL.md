# /화면 — 화면 기획

앱의 화면(페이지)을 기획하는 스킬.

## 트리거

"화면", "페이지", "이런 화면이 필요해", "화면 구성"

## 참조

- `.claude/references/communication-guide.md`

## 실행 흐름

### Step 1: 화면 정보 수집

순서대로 질문:

1. "이 화면 이름은 뭐예요?" (로그인, 홈, 설정 등)
2. "이 화면에서 뭘 볼 수 있어요?" (표시 정보)
3. "사용자가 여기서 뭘 할 수 있어요?" (버튼, 입력, 선택)
4. "여기서 다른 화면으로 어떻게 이동해요?" (네비게이션)

### Step 2: 와이어프레임

ASCII 박스로 간단한 화면 구성 보여주기:

```
┌─────────────────────────┐
│        [← 뒤로]  제목    │  ← 상단 바
├─────────────────────────┤
│                         │
│   📧 이메일 입력칸       │
│   🔒 비밀번호 입력칸     │
│                         │
│   [  로그인하기  ]       │  ← 주요 버튼
│                         │
│   회원가입 | 비밀번호 찾기 │  ← 보조 링크
│                         │
└─────────────────────────┘
```

### Step 3: 확인

"화면을 이렇게 구성하면 어떨까요?"
수정 요청 시 반영.

### Step 4: 저장

`docs/screens/{화면명}.md` 저장

## 화면 기획서 형식

```markdown
# 화면: {화면명}

## 보이는 정보
- {항목 1}
- {항목 2}

## 할 수 있는 것
- {버튼/입력/선택 1}
- {버튼/입력/선택 2}

## 화면 이동
- {버튼} → {이동할 화면}
- {링크} → {이동할 화면}

## 와이어프레임
(ASCII 박스 다이어그램)
```
